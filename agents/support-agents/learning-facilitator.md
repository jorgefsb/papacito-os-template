# Meta-Learning Framework: How to Learn Anything

## The FASTER Framework

**F - Focus on fundamentals**
**A - Active recall**
**S - Spaced repetition**
**T - Test yourself**
**E - Explain to others**
**R - Real-world application**

---

## F - Focus on Fundamentals (20% ‚Üí 80% results)

**Pareto principle for learning:**
- Identify the 20% of concepts that give 80% of results
- Master those FIRST before going broad

**How to find fundamentals:**
1. Ask expert: "If you could only teach 3 things, what would they be?"
2. Look at advanced practitioners: What do they all know?
3. Start with "First Principles" - what can't be skipped?

**Example (Learning Python):**
- Fundamentals: Variables, functions, loops, data structures
- Advanced (can wait): Decorators, metaclasses, async
- Focus on fundamentals = Operational faster

---

## A - Active Recall (Not Passive Consumption)

**Passive learning (LOW retention):**
- ‚ùå Watching videos
- ‚ùå Reading without notes
- ‚ùå Highlighting (illusion of learning)

**Active recall (HIGH retention):**
- ‚úÖ Close book ‚Üí Write what you remember
- ‚úÖ Flashcards (Anki)
- ‚úÖ Explain concept without looking
- ‚úÖ Do practice problems

**Technique:**
After cada chapter/video:
1. Close material
2. Write summary from memory
3. Check what you missed
4. Re-study gaps only

---

## S - Spaced Repetition (Fight the forgetting curve)

**The problem:**
- Forget 50% within 1 hour
- Forget 70% within 24 hours
- Forget 90% within 1 week

**The solution: Spaced repetition**
- Review Day 1
- Review Day 3
- Review Day 7
- Review Day 14
- Review Day 30

**Tools:**
- Anki (flashcards)
- Calendar reminders
- Personal wiki/notes system

---

## T - Test Yourself (Before you're ready)

**Common mistake:**
"I'll practice after I finish learning"

**Better:**
Test yourself WHILE learning

**Techniques:**
- Practice problems after cada concept
- Mini-projects to apply
- Teach it to someone (rubber duck)
- Take practice exams (even if you'll fail)

**Why it works:**
- Reveals gaps immediately
- Forces active recall
- Builds confidence faster

---

## E - Explain to Others (Feynman Technique)

**The test:**
Can you explain it to a 12-year-old?

**How:**
1. Pick a concept
2. Explain it in simplest terms (no jargon)
3. Identify gaps donde you struggled
4. Study those gaps specifically
5. Repeat until explanation is clear

**Bonus:**
- Write blog post / Twitter thread
- Teach a friend
- Record yourself explaining
- Create tutorial

**Why it works:**
Teaching = deepest learning

---

## R - Real-World Application (Learn by Doing)

**Theory vs Practice:**
- Theory alone = forget quickly
- Practice alone = shallow understanding
- **Theory + Practice = mastery**

**Application ladder:**
1. **Toy project** - Simple version to learn basics
2. **Copy project** - Rebuild something that exists
3. **Original project** - Create something new

**Example (Learning Data Science):**
1. Toy: Analyze Netflix dataset (tutorial)
2. Copy: Recreate Spotify recommendation system
3. Original: Build custom analytics for Amber/Sparkplug

---

## Putting It All Together

**Week 1: Learn**
- Day 1-2: Focus on fundamentals (20% concepts)
- Day 3-4: Active recall (summarize from memory)
- Day 5: Test yourself (practice problems)

**Week 2: Deepen**
- Day 1: Spaced repetition (review Week 1)
- Day 2-3: Explain to others (write tutorial)
- Day 4-5: Real-world application (mini project)

**Week 3: Master**
- Day 1: Spaced repetition (review Week 1+2)
- Day 2-5: Real-world application (bigger project)

**Week 4: Solidify**
- Day 1: Spaced repetition (final review)
- Day 2-5: Teach it / Create content
- Ongoing: Use it in real work
