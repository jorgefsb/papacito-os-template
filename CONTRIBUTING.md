## How to Contribute

1. **Fork the repository**
2. **Create a branch** (`git checkout -b feature/amazing-agent`)
3. **Make your changes**
4. **Test your changes**
5. **Commit** (`git commit -m 'Add amazing agent'`)
6. **Push** (`git push origin feature/amazing-agent`)
7. **Open a Pull Request**

## Guidelines

### Code Style
- Follow the existing structure and naming conventions
- Use markdown for all documentation
- Keep agents modular and focused on single responsibility

### Documentation
- Document your agents clearly with examples
- Provide real-world use cases
- Update the README if adding major features

### Agent Creation
- Use the agent template in `/templates/agent-template.md`
- Define clear activation keywords
- Specify collaboration patterns with other agents
- Include example inputs and outputs

### Testing
- Test your agent in isolation
- Test collaborations with existing agents
- Validate that Router detects your agent correctly

## Commit Messages

Use [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add Content Strategist agent
fix: correct Router detection logic
docs: update architecture documentation
refactor: improve collaboration protocol
test: add validation for agents
chore: update dependencies
```

## Pull Request Process

1. Update the README.md with details of changes
2. Update the CHANGELOG.md with a note about your change
3. The PR will be merged once you have the sign-off of maintainers
